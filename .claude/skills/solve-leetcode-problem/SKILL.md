# Regenerate LeetCode Solutions

You are already an AI agent. This skill works under both Codex and Claude. Read the problem files, generate solutions yourself, verify them, and write the results to disk. No external LLM CLI calls are needed unless the user explicitly asks for that tool-driven workflow.

## Model and output target

Always use the strongest available model for the current provider:

| Agent | Strongest model | Output file |
|---|---|---|
| Claude | `claude-opus-4-8` | `solution-claude-opus.md` |
| Codex | `gpt-5` (or latest available) | `solution-codex.md` |

- If the user asks for a specific provider, use that provider's strongest model.
- If the user names an exact output file, use that file.
- If the user does not specify a provider, default to the current agent's strongest model.

Set the header's `Generated by` value to the exact model name (e.g. `claude-opus-4-8`, `gpt-5`). Do not overwrite the other provider's solution file unless the user explicitly asks.

## Step 1 — Determine scope

Read `problem-index.csv` (columns: `Number,Title,Difficulty,Acceptance,Category,URL`) to find matching problems:

- All problems → every row
- Specific numbers: e.g. "regenerate 1 and 3" → filter by Number
- By difficulty: e.g. "only Medium" → filter by Difficulty column
- By category: e.g. "Array problems" → filter by Category column

## Step 2 — For each problem, generate the solution

### 2a. Derive the problem directory

Slug is the last path segment of the URL:
- `https://leetcode.com/problems/two-sum/` → slug `two-sum`
- Directory: `problems/{Number}-{slug}/`

### 2b. Read the problem

Read `problems/{Number}-{slug}/problem.md`.

### 2c. Find reference solutions (optional)

Check `/Users/hafizahsan/projects/github/hafiz-ahsan/grind75` for a subdirectory matching `{Number}-*`. If found, read any `.py`, `.java`, `.cpp`, `.js`, or `.md` files inside — use them to inform your solution.

### 2d. Generate the solution

Using your own understanding, produce a solution that contains:

1. **Intuition** — the key insight or pattern that makes this problem solvable
2. **Approach** — step-by-step pseudocode
3. **Python Solution** — clean, heavily-commented implementation. Comment the *reasoning* behind each step, not just what it does.
4. **Complexity Analysis** — time and space, with justification
5. **Alternative Approaches** — briefly note other valid approaches

### 2e. Verify with uv run

Extract the Python `Solution` class, parse the input/output examples from `problem.md`, write a temp script, and run it:

```bash
uv run python /tmp/verify_{Number}.py
```

The script should instantiate `Solution`, call the method with each example's input, and assert the output matches. Include minimal stubs for `ListNode` / `TreeNode` if needed.

- Exit 0 → proceed
- Failure → fix your solution and retry, up to 2 more times
- Can't parse code → skip verification, note it in the summary

### 2f. Write the output file

Write to the selected output target:

```text
problems/{Number}-{slug}/{output_filename}
```

Common targets:

- Codex: `problems/{Number}-{slug}/solution-codex.md`
- Claude: `problems/{Number}-{slug}/solution-claude-opus.md`

**The file must always begin with this header:**

```markdown
# Solution for {Number}. {Title}

**Generated by:** {your model name}
**Date:** {YYYY-MM-DD HH:MM:SS}
**Source:** <{URL}>

---

{solution content}
```

## Step 3 — Confirm

Report for each problem:
- ✅ Generated and verified
- ⚠️ Generated but verification failed after 3 attempts (flagged for review)
- ⚠️ Generated but verification skipped (could not parse code)
- ❌ Skipped (problem.md not found)

Include total counts.
